# Development update-signing keys

This directory holds an Ed25519 keypair used by the Tauri auto-updater
during local development. The public key is also embedded in
`app/src-tauri/tauri.conf.json` under `plugins.updater.pubkey` so the
desktop app trusts updates signed by the matching private key.

## What this is NOT

**Not a production key.** It is committed to the repository and shared
across every clone. Anyone who can read the repo can sign update bundles
that any locally-built copy of Nomi will accept. That is intentional —
it lets contributors test the auto-update flow end-to-end without
coordinating with a maintainer — but it means the dev key cannot be
used to ship updates to real users.

## Production key custody

The production update-signing keypair is generated by the maintainer
out-of-band, kept in 1Password (or equivalent), and exposed to CI as
`TAURI_SIGNING_PRIVATE_KEY` (string) + `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
GitHub Actions secrets. The `pubkey` value in `tauri.conf.json` is
flipped to the production public key as part of the first signed release
build.

Rotation, leaked-key, and renewal procedures (including the dev → prod
key cutover, which forces every existing install to be reinstalled)
live in [`docs/runbooks/signing-certs.md`](../docs/runbooks/signing-certs.md).

## Local usage

To sign a fixture update bundle for testing:

```bash
TAURI_SIGNING_PRIVATE_KEY="$(cat dev-keys/nomi-dev.key)" \
TAURI_SIGNING_PRIVATE_KEY_PASSWORD="" \
  npx tauri build
```

Tauri's bundler picks up the env vars and writes a signed `latest.json`
manifest beside the artifact. The desktop app, built from this
repository, will trust that signature because the embedded `pubkey`
matches.

## Regenerating the dev keypair

If the dev keypair ever needs to be rotated (e.g. accidentally pasted
to a public channel — though there is nothing sensitive about it),
regenerate via:

```bash
cd app
CI=1 npx tauri signer generate --ci -p "" \
  -w ../dev-keys/nomi-dev.key -f
```

Then copy the contents of `dev-keys/nomi-dev.key.pub`, strip newlines,
and paste as the value of `plugins.updater.pubkey` in
`app/src-tauri/tauri.conf.json`.
